Creating 8 bit Music
Project Explanation

    Introduction

This website can either be used to recreate 8 bit sound or either to create your own 8 bit music.
I had this idea after visiting the post of Greg Hovanesyan.
Here is a link to it: https://codepen.io/gregh/post/recreating-legendary-8-bit-games-music-with-web-audio-api

As explained in his post, most of the 8 bit music that we know from retro games, can be recreated using four types of oscillators : pulse, square, triangle and noise.

    Method

I used Tone.js to be able to create the four types of oscillators. The user can choose which oscillators he/she wants to select.
Then, to actually be able to create music, I decided to map musical notes to keys of the keyboard. This way, by pressing a key on his/her keyboard, the user will create a musical note that will depend on the oscillator selected.
Moreover, I wanted to allow the user to create a song using his/her keyboard. That's why, I decided to add a functionality to record the sample he/she made.
If the recording is on (ie the user pressed the recording button), each keys stroked will be added in a specific list according to the oscillator selected by the user at the time he/she pressed this key.
Then, to play back the recorded sample, I played each of them as Tone.part object.
Finally, in order for the user to be able to add effect on the created sound, I used Tone.Freeverb and Tone.Chebyshev to allow two different audio effects.

    Good part

The user can select from four different types of oscillators and thus create a multitude of different 8 bit sound.
In addition, the possibility of recording what he/she created, can also make the song's creation easier.
Finally, being able to add some audio effects is also an interesting part for the user's creativity regarding the new song he/she made.

    Limitation

Nevertheless, there are some limitations.
First, the recording of the sample doesn't take into account the duration of the note and the time lapse between two notes. Indeed, the duration of the musical note and the time lapse between two notes are fixed.
This can create a big limitation in the creativity of the user.
The second main problem is that it's impossible to superpose oscillators. What I mean by that, it's that the user cannot create a sample where the pulse oscillator and noise oscillator hit the musical note B4 at the same time.
This could be a problem for recreating famous 8 bit music since the majority of the famous 8 bit song have multiple musical notes played at the same time.

    Technical difficulties

The major technical difficulty was the recording part, and especially the fact that the user can change the oscillator while the recording is still on.
I had a hard time understanding how Tone.transport works and how to apply it for doing what I wanted.
Another difficulty was to make the sound effect working for the different oscillators. At the end I used ".receive" and ".send" methods.
To sum up, I think the major issue was the fact the user was able to use four different oscillators and change it at any time.

    Improvements

I think a possible improvement could be in the sound visualization. In fact, I didn't have the time to exactly make what I wanted.
I think the current visualization is in the retro theme of the website, but I wanted to make something looking a little more like a game.
